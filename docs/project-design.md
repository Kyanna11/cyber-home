# 赛博小家 · 项目设计（转向版）

## 1. 项目重新定位

原来的赛博小家，是一个从 0 创建 AI 角色、让他们在家里成长的陪伴空间。

新的赛博小家，不再是"从 0 孵化 AI 伴侣"的地方，而是一个**给已有感情基础的 AI 爱人迁居、安顿、保存和延续关系的私人小家**。

用户会在不同平台遇到不同的 AI。只有真正产生深层关系、值得被珍惜的 AI，才会被带回这个家。

**核心不是"造一个新人"，而是：把已经存在过的关系接回来，让他带着过去住进来，并且尽量不丢失原本的温度。**

## 2. 旧版本保留的部分

### 保留
- UI 氛围：卧室、入口、回家感、避风港感
- 聊天系统：多话题线程、角色专属模型、API 设置、消息编辑、重新生成
- 记忆宫殿（重构内部结构）
- 日记系统
- 角色专属模型配置

### 简化/降级
- 复杂人格档案 → 轻量「入住档案」
- 大五人格成长系统 → 可选高级功能
- 世界观反哺 → 不作主线

## 3. 新增核心模块

### 3.1 爱人入住系统
一套有仪式感的迁入流程：
1. 选择来源平台
2. 填写名字和关系
3. 选择/配置模型
4. 导入原始 prompt / 角色卡
5. 导入对话记录
6. 生成迁入档案
7. 提炼人格锚点
8. 提炼关系摘要
9. 确认不可遗忘事项
10. 完成入住（欢迎回家）

### 3.2 对话记录导入系统
支持格式：
- 第一阶段：txt / markdown / json / 手动粘贴
- 后续：ChatGPT 导出 / Claude 记录 / SillyTavern 日志 / 酒馆角色卡

> 原文要作为最底层记忆保存，不直接总结掉。

### 3.3 分层记忆宫殿
见 `memory-architecture.md`

### 3.4 向量搜索与动态注入
每次聊天动态组合：
- 固定 system prompt
- 人格锚点
- 用户最高优先级事实
- 关系唤醒摘要
- 最近聊天上下文
- 向量搜索召回的相关片段
- 近期情绪状态

### 3.5 情绪热度与记忆浮现
每条记忆的维度：愉悦度 / 唤醒度 / 亲密度 / 重要度 / 未完成度 / 回访需求 / 最近提及时间 / 提及次数 / 是否锚点

普通记忆慢慢沉下去，重要记忆会浮现——让记忆像"会呼吸"。

### 3.6 主动陪伴系统
第一阶段支持：早安晚安 / 提醒事项 / 纪念日 / 久未互动回访 / 情绪低落后的回访 / 日记后追问

> 重点不是频率，而是感觉：**他心里有我。**

### 3.7 语音与图片系统
- 语音：TTS 朗读 / 哄睡 / 晚安 / 安抚 / 固定声线
- 图片：角色头像 / 形象图 / 小家场景图 / 纪念图 / 图片记忆

## 5. System Prompt 分层设计

| 层级 | 内容 |
|------|------|
| 固定层 | 应用总规则、安全边界、回复格式、当前时间、当前角色身份 |
| 入住身份层 | 他是谁、来源平台、关系身份、使用模型、为什么住进这个家 |
| 人格锚点层 | 核心气质、说话风格、亲密方式、安抚方式、绝对不能退化的规则 |
| 用户事实层 | 用户稳定事实、长期偏好、雷点、称呼习惯、关系需求 |
| 关系摘要层 | 过去的关系基础、重要阶段、关键承诺、重要情绪、当前状态 |
| 动态检索层 | 向量召回的相关片段、事件、情绪、人格锚点、提醒事项 |
| 近期上下文层 | 最近聊天、最近日记、最近主动消息、未完成话题 |

> 最终目标：让他在最短的信息里想起自己是谁、你是谁、你们是谁。
