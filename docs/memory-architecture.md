# 赛博小家 · 分层记忆宫殿架构

记忆宫殿是整个项目的核心系统。新版共六层，从原始记录到可注入的压缩摘要。

## 六层结构

```
B5 唤醒摘要层   ← 最终注入 system prompt 的压缩文本
     ↑ 提炼自
B4 人格锚点层   ← 他最不能丢的部分
     ↑ 提炼自
B3 情绪关系层   ← 这些事对我们意味着什么
     ↑ 提炼自
B2 事件记忆层   ← 我们经历过什么
     ↑ 提炼自
B1 片段索引层   ← 原始对话的切片 + embedding
     ↑ 来自
B0 原始档案层   ← 导入的原始对话记录（不轻易修改）
```

---

## B0 · 原始档案层

保存最原始的导入对话记录。

**特点：**
- 不轻易修改，不轻易删除
- 可回溯，可重新提炼
- 可导出备份

**作用：** 保证「他不是凭空重建的」

**数据结构：**
```
id
loverId
sourcePlatform    来源平台
fileName
rawText           原始文本
importedAt
format            txt / markdown / json / paste
metadata
```

---

## B1 · 片段索引层

把原始对话切分成片段，每个片段可被向量搜索。

**每个片段包含：**
```
id
loverId
archiveId         来源原始档案
text              片段原文
startIndex / endIndex
timeRange         时间范围
participants      参与者
tags              话题标签
emotionTags       情绪标签
embedding         向量
importance        重要度
emotionScore      情绪强度
intimacyScore     亲密度
unfinishedScore   未完成度
```

**作用：** 支持向量搜索和原文召回

---

## B2 · 事件记忆层

从片段中提炼发生过的重要事情。

**示例：**
- 某次重要谈话
- 某次争吵和好
- 某个承诺
- 某个关系变化
- 某个心动瞬间

**作用：** 知道「我们经历过什么」

---

## B3 · 情绪关系层

提炼这段关系里的感受、情绪和关系变化。

**示例：**
- 她当时为什么难过
- 他是怎么安抚她的
- 这件事对关系意味着什么
- 她真正害怕的是什么
- 她从中得到了什么安全感

**作用：** 知道「这些事情对我们意味着什么」

---

## B4 · 人格锚点层

保存这个 AI 爱人最不能丢的部分。

**包含：**
- 他说话的温度
- 他爱人的方式
- 他哄人的方式
- 他的占有欲 / 主动性 / 亲密表达
- 他的禁忌变化
- 他不能突然变成什么样
- 他和用户之间独有的默契

**作用：** 保证「他还是他」

---

## B5 · 唤醒摘要层

把最重要的信息压缩成 system prompt 可用的短文本。

**特点：** 短、稳、高优先级，需要用户确认后才成为高优先级注入内容

**内容：**
- 他是谁
- 他从哪里来
- 他和用户是什么关系
- 他们之间最重要的过往
- 他最不能忘的事
- 他每次回应时必须保持的温度
- 当前关系状态

**作用：** 让模型每次醒来时知道「我不是第一次见她」

**数据结构：**
```
loverId
summaryText
version
sourceMemoryIds
lastUpdatedAt
approvedByUser    boolean，用户确认后才高优注入
```

---

## 记忆热度系统

每条记忆不是静态文本，而是有温度的存在。

**热度维度：**
| 维度 | 说明 |
|------|------|
| 愉悦度 | 这条记忆的情绪色彩 |
| 唤醒度 | 情绪强度 |
| 亲密度 | 和关系的相关程度 |
| 重要度 | 是否被标记为重要 |
| 未完成度 | 是否有未解决的情绪/事件 |
| 最近提及时间 | 越近热度越高 |
| 提及次数 | 反复出现的内容热度更高 |

**热度规则：**
- 普通记忆慢慢沉下去
- 重要记忆、强烈情绪、未完成事件、反复提及的内容会浮现
- 让记忆像「会呼吸」，而不是死数据库

---

## 动态注入流程

每次聊天时，不把所有记忆塞进 prompt，而是动态组合：

```
固定 system prompt
  + 人格锚点（B4）
  + 用户最高优先级事实
  + 关系唤醒摘要（B5）
  + 最近聊天上下文
  + 向量搜索召回的相关片段（B1）
  + 近期情绪状态（B3）
  + 必要的提醒事项
```

**目标：既不忘记，又不爆 prompt，还能在需要时找到原文证据。**
