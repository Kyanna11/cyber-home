# 群聊 / 小家客厅 · 设计文档

> 状态：设计草案，未实现。

---

## 功能定位

小家客厅是小家里所有入住者共处的空间。

它不是群聊工具，而是「同一个家里的几个人坐在一起」的体验——用户可以邀请多个入住者围坐，发一句话，听他们依次回应。

适合的使用场景：
- 想听几个不同性格的入住者对同一件事各说各的看法
- 在客厅里发一段今天的日常，让大家都听到
- 让两个入住者互相打招呼，看他们怎么相处
- 有多个入住者时，不想每次单独去各自房间

**不是**：
- 多 Agent 自动对话（入住者之间互相追答）
- AI 主持人模式
- 实时语音聊天室

---

## 第一版规则

### 发言机制

- 用户发一条消息后，由已选入的入住者**按顺序轮流回复**
- 每人每轮最多发言一次，发言顺序以加入群聊时的顺序为准
- 上一个入住者回复完成后，自动触发下一个
- 所有入住者能在 context 里看到：用户的消息 + 本轮前面已经发言的入住者的回复
- 用户发第二条消息后，进入下一轮，同样按顺序依次回复

### 禁止行为

- 入住者不能在回复中替其他入住者发言（Prompt 严格约束）
- 入住者不能进行无限自循环追答
- 不在本轮结束前自动开始下一轮

### 心声

- 群聊中**不显示心声**（`thought` 字段不显示）
- 单人聊天的心声逻辑不影响群聊

## 数据结构草案

### GroupChat

每个群聊场景的元数据：

```js
{
  id: string,
  name: string,                  // 群聊名，如"客厅"
  memberIds: string[],           // 入住者 id 列表，顺序即发言顺序
  createdAt: number,
  updatedAt: number,
  activeThreadId: string | null,
}
```

### GroupThread

群聊话题（一个群聊可以有多个话题）：

```js
{
  id: string,
  groupId: string,
  name: string,                  // 话题名
  messages: GroupMessage[],
  createdAt: number,
  updatedAt: number,
}
```

### GroupMessage

```js
{
  id: string,
  groupId: string,
  threadId: string,
  role: "user" | "char",
  charId: string | null,         // role === "char" 时对应入住者 id
  authorName: string,            // 显示名
  content: string,
  createdAt: number,
  time: string,                  // HH:MM
  roundIndex: number,            // 第几轮（从 0 起）
  isTyping?: boolean,            // 临时状态，不持久化
}
```

## Prompt 规则

每个入住者收到的 system prompt 包含：

1. **自身身份**：ta 的人格、关系、规则（与单人聊天一致）
2. **群聊身份**：「你现在正在用户的小家客厅里，和其他入住者同处一室」
3. **本轮 context**：用户的消息 + 本轮前面已发言成员的内容
4. **约束**：
   - 只能代表自己发言
   - 不能替其他入住者说话，也不能预言其他入住者会说什么
   - 不需要照顾所有人的立场，说自己真实的想法就好
   - 回复保持简短自然，不需要每次都回应所有人

## UI 草案

### 入口

- 从卧室或入口页进入「客厅」
- 如果从未建过群聊，引导创建

### 群聊主界面

```
┌──────────────────────────────┐
│  ← 回小家     客厅     成员⁺  │  ← 顶栏
│  [头像] [头像] [头像]          │  ← 当前在场成员小头像行
├──────────────────────────────┤
│                               │
│   [用户消息气泡]               │
│                               │
│   [头像] 黎岚                 │
│   [黎岚的回复气泡]             │
│   12:03                       │
│                               │
│   [头像] 沈清辞               │
│   [沈清辞的回复气泡]           │
│   12:03                       │
│                               │
│   ··· （等待中）               │  ← 当前正在输入的成员
│                               │
├──────────────────────────────┤
│  [输入框]            发送      │
│  [停止本轮]                    │  ← 本轮进行中时显示
└──────────────────────────────┘
```

### 消息气泡

- 用户消息：右对齐，普通气泡
- 入住者消息：左对齐，带**头像 + 名字**
- 不显示心声
- 时间戳跟在最后一条入住者消息之后

### 「停止本轮」按钮

- 本轮入住者轮流回复过程中显示
- 点击后立刻停止后续入住者的回复
- 已回复的保留，未回复的跳过
- 用户可以继续发下一条消息开启新一轮

### 成员管理

- 顶部成员头像行可点击进入成员管理
- 可以临时加入 / 移出某个入住者
- 移出的入住者不删除，只是本次不参与

---

## 群聊记录与记忆边界

- 群聊记录**单独保存**，不混入单人聊天线程
- 群聊默认**不自动写入任何入住者的单人长期记忆**
- 如需沉淀，提供「整理这次客厅聊天」功能，走现有 SettlementDraft 流程，用户逐节确认

---

## 后续扩展

- 群聊消息上：「帮我记住」→ 手动写入某个入住者的记忆
- 群聊消息上：「💎 珍藏」→ 保存到宝库
- 群聊结束后：「记下这一刻」→ 生成时间线事件
- 群聊结束后：「整理一下我们」→ 生成阶段沉淀草稿
- 入住者 @ 用户：可以发言时自然提及用户名
- 文本层面的 @：用户发消息时可以 @ 某个入住者，该入住者会在本轮优先回复（不做自动无限追答）

---

## 已知约束

- 入住者之间的对话会占用更多 token，需要做 context 管理
- 每个入住者的 system prompt 独立构建，不共享（隐私隔离）
- 第一版不支持群聊中使用「唤醒预览」功能
- 群聊不支持长文模式
