# 亲密邀请 / 场景会话 · 设计文档

> 状态：设计草案，未实现。  
> 入口已预留：聊天页加号面板「发出亲密邀请」按钮（当前为 disabled）。

---

## 功能定位

亲密邀请不是普通聊天的延伸，而是一次**有仪式感的场景进入**。

用户可以为当前入住者设置一个具体的场景——时间、地点、氛围、前情——然后发出邀请，由入住者先开口，带用户进入这个场景。

适合的使用场景：
- 想在某个特定氛围里和入住者待一会儿（深夜、雨天、咖啡馆）
- 想重温某个虚构的约定或未竟的故事
- 给自己一个短暂脱离现实的陪伴时刻
- 有特定创作想法，想让入住者先开口带入

**不是**：
- 普通聊天的换皮
- 角色扮演游戏引擎
- 需要 AI 自动续写剧情的自动化功能

---

## 邀请表单字段

用户填写以下信息后发出邀请：

| 字段 | 类型 | 说明 |
|------|------|------|
| `invitedCharId` | string | 邀请的入住者 id（默认当前入住者） |
| `scene` | string | 场景描述，如「深夜十一点的公寓，刚下过雨」 |
| `mood` | string | 氛围标签，如「安静」「温柔」「有一点难过」 |
| `context` | string | 前情提要，如「我们刚分开了两周，你刚回来」 |
| `invitation` | string | 邀请内容，如「我想和你待一会儿」 |
| `replyStyle` | string | 回复方式：`"chat"` 普通 / `"long"` 长文 |

所有字段可选，但场景（`scene`）和邀请内容（`invitation`）建议至少填一个。

## 数据结构草案

### SceneConfig

保存本次场景的配置，挂在 SceneThread 上：

```js
{
  invitedCharId: string,
  scene:      string,
  mood:       string,
  context:    string,
  invitation: string,
  replyStyle: "chat" | "long",
  startedAt:  number,
}
```

### SceneThread

场景是独立的 thread，与普通话题隔离：

```js
{
  id: string,
  charId: string,
  type: "scene",                 // 区分普通 thread 和 scene thread
  name: string,                  // 如 "深夜的公寓 · 和黎岚"
  sceneConfig: SceneConfig,
  messages: SceneMessage[],
  createdAt: number,
  closedAt: number | null,       // null = 场景进行中，timestamp = 已结束
}
```

### SceneMessage

与普通 message 结构基本一致，额外标记 scene context：

```js
{
  id: string,
  role: "user" | "bot",
  content: string,
  time: string,
  replyMode: "chat" | "long",
  isOpeningMessage?: boolean,    // 入住者的开场第一句，true
  // thought 不显示，但仍可存储
  thought?: string,
}
```

### Treasure.type = "scene"

场景收藏到宝库时，使用 `type: "scene"`：

```js
{
  // ... 普通 Treasure 字段
  type: "scene",
  sceneConfig: SceneConfig,      // 保存场景配置，方便以后重新打开
}
```

## 场景开始流程

1. 用户填写邀请表单，点击「进入场景」
2. 创建独立 SceneThread，记录 `sceneConfig`
3. 进入场景界面（见 UI 草案）
4. **由入住者先发言**——系统用 sceneConfig 构建 Prompt，自动发起一次 LLM 调用
5. 入住者的第一句话（`isOpeningMessage: true`）显示后，等待用户输入
6. 后续对话正常进行，直到用户点击「今天就到这儿」

---

## Prompt 规则

场景 Prompt 在入住者原始 system prompt 基础上追加场景 context：

```
【场景设定】
以下是此刻的场景：{scene}
氛围：{mood}
前情：{context}

{invitation}

请从现在开始进入这个场景，自然地先开口说第一句话。
不要解释你在做什么，不要说"好的"，直接进入场景。
```

- 普通聊天的 system prompt 逻辑（人格、记忆注入等）保持不变
- 场景 Prompt 追加在其后
- `thought`（心声）在场景中依然生成，但**默认不显示**

场景界面与普通聊天的**视觉区分**：
- 背景色不同（更深、更暗，有沉浸感）
- 顶部不显示在线状态 / 模型名
- 不显示「更多菜单」中的清空/导出等功能
- 不显示心声（`thought` 不渲染）
- 底部「今天就到这儿」按钮替代/补充普通聊天的其他操作

## 结束场景

点击「今天就到这儿」后弹出结束选项：

```
┌────────────────────────┐
│  今天就到这儿了。        │
│  想把今晚留下来吗？      │
├────────────────────────┤
│  只是结束，不保存        │
│  💎 收藏整段到我的宝库   │
│  🕰 记下这一刻           │
│  🌙 把今晚轻轻收好       │  ← 三合一快捷操作（宝库+时间线）
└────────────────────────┘
```

选项说明：

| 选项 | 行为 |
|------|------|
| 只是结束 | 关闭场景，不做任何写入 |
| 收藏整段到我的宝库 | 把整段 SceneThread 作为 `type: "scene"` 存入宝库，用户可填标题 |
| 记下这一刻 | 进入时间线事件创建流程，预填标题/内容 |
| 把今晚轻轻收好 | 同时存宝库 + 创建时间线事件，一步完成 |

---

## 记忆边界

- 场景对话**默认只保存原文**，不自动提取记忆、不自动写入长期记忆
- 如需沉淀，用户可以手动触发「整理一下我们」，走现有 SettlementDraft 流程
- 场景内的对话不会被主聊天线程的「帮我记住」功能影响

## 后续扩展

- **场景模板**：预置几个常见场景（深夜、旅途中、清晨等），一键填入
- **自定义场景**：保存用户自己创建的场景配置，方便下次直接打开
- **从宝库继续打开场景**：点击宝库中 `type: "scene"` 的条目，可以用原 sceneConfig 重新进入场景
- **场景内心声切换**：用户可以手动打开心声显示
- **场景氛围声**：未来 TTS / 音效阶段的预留位置
