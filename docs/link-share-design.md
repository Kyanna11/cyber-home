# 分享链接 / 给他看这个 · 设计文档

> 状态：设计草案，未实现。  
> 入口已预留：聊天页加号面板（当前为 disabled）。

---

## 功能定位

用户可以把外部链接分享给当前入住者——比如一篇文章、一个视频、一个帖子——然后和 ta 聊聊这个内容。

它模拟的是「给对方发了一个链接，然后问 ta 觉得怎么样」的体验。

适合的场景：
- 看到一篇触动你的文章，想听 ta 说说
- 发现一个和 ta 相关的内容，想分享
- 想基于某个话题背景聊，而不是凭空开口

---

## 第一版规格

### 不做的事（明确边界）

- **不做**复杂网页抓取（不 fetch URL 内容）
- **不做** summary API 调用
- **不假装** ta 真的能看到链接里的内容
- **不支持**微信/抖音/B站私链等需要登录的内容

### 能做的事

- 用户填写链接、标题、备注、分享意图
- 以链接卡片形式发送到聊天
- Prompt 告知入住者：「用户分享了一个链接，ta 可能无法直接访问，只能根据用户提供的描述回应」
- 入住者基于**用户的标题和备注**来回应，不编造链接里没有的内容

---

## 分享表单字段

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `url` | string | 是 | 链接地址 |
| `title` | string | 否 | 用户填写的标题或简短描述 |
| `note` | string | 否 | 备注：「我觉得很触动」等 |
| `intent` | string | 否 | 分享意图（见下表） |

分享意图选项（与手札分享意图一致）：

| 值 | 显示文案 |
|----|---------|
| `read` | 只是给你看看 |
| `react` | 想听你说说感受 |
| `discuss` | 想一起聊聊 |
| `remember` | 希望你记住这个 |

---

## 消息数据结构

链接分享以 user 消息发送，带特殊标记：

```js
{
  role: "user",
  content: title || url,         // 发给 LLM 的实际 content（见 Prompt 规则）
  isLinkShare: true,
  linkUrl:   string,
  linkTitle: string,
  linkNote:  string,
  linkIntent: string,
  time: string,
}
```

---

## UI：链接卡片（消息区）

链接分享不显示为普通气泡，而是一张链接卡片：

```
┌─────────────────────────────┐
│  🔗  这篇文章很触动我          │  ← 标题
│                               │
│  https://example.com/...      │  ← URL（截断显示）
│                               │
│  备注：最后一段让我想到了你    │
├─────────────────────────────┤
│  想一起聊聊 · 12:34           │  ← 分享意图 + 时间
└─────────────────────────────┘
```

---

## Prompt 规则

构建发给 LLM 的 user 消息时，将链接卡片转换为文字描述：

```
[用户分享了一个链接给你。注意：你可能无法直接访问这个链接，也无法看到页面内容。请只根据用户提供的标题和描述来回应，不要编造你没有读到的内容，不要假装看过了，不要说"我看了这篇文章"。]

链接：{url}
标题：{title}
{note ? "用户备注：" + note : ""}
{intent 对应的提示文字}
```

`intent` 对应的提示追加：
- `react`：「用户想听你分享一下自己的感受或想法」
- `discuss`：「用户想基于这个话题和你聊聊」
- `remember`：「用户希望你以后记得这个内容，请在回复中认真确认你记住了」
- `read`（默认）：不追加额外提示

## 后续扩展

- **链接 → 手札**：把链接卡片及备注一键保存为一篇手札草稿
- **链接 → 宝库**：把入住者的回复和原链接一起收藏到宝库
- **链接 → 时间线**：把「分享了 xxx 给 ta」记录为时间线事件
- **链接 → 声声档案草稿**：如果链接内容触发了对用户的理解，提炼到声声档案
- **摘要辅助**：未来可以考虑提供「让用户自己粘贴摘要」的输入框，不做自动抓取
