# 他的宝库 + 我们的照片回忆 · 设计文档

> 状态：设计草案，均未实现。  
> 现有「我的宝库」是用户维度的，「他的宝库」是入住者维度的，两者独立。

---

## 一、他的宝库

### 定位

「我的宝库」是用户珍藏 ta 说过的话。  
「他的宝库」是**入住者珍藏用户**的话、手札、重要片段。

这不是镜像关系，而是视角的翻转——他在意什么，他在收集什么，他想记住什么。

---

### 第一版功能

#### 用户主动让 ta 珍藏

用户可以在自己的消息上触发「让 ta 珍藏」操作：
- 入口：聊天页用户消息右下角，点击「💝 让 ta 珍藏」
- 操作后：将该消息保存到**当前入住者的宝库**里，不写入 prompt，不触发任何记忆流程
- 可以填写备注：「我希望 ta 记得这句话」

#### 数据结构草案

```js
// CharTreasureItem
{
  id: string,
  charId: string,                // 属于哪个入住者
  sourceType: "message" | "note" | "manual",
  sourceId: string | null,       // 消息 id / 手札 id
  content: string,               // 保存的正文内容
  note: string,                  // 用户备注
  authorType: "user",            // 第一版只允许 user 内容
  createdAt: number,
  pinned: boolean,
}
```

#### 不做的事（第一版边界）

- 不自动注入到 prompt（不影响聊天行为）
- 不自动写入记忆宫殿
- 入住者不会主动说「我收藏了这句话」（需要用户手动查看）
- 不做 AI 自动建议收藏（这是后续功能）

---

### 后续扩展

- **入住者主动提出珍藏**：入住者在回复中可以自然表示「我想记住这句话」，系统检测后弹出确认，用户点击确认才写入
- **手札 → 他的宝库**：在手札编辑页，可以「分享到某入住者的宝库」
- **他的宝库可查看**：在「他的房间」中，有一个「他的珍藏」入口，展示他收藏的所有内容
- **宝库反哺 prompt**：未来可以选择性将宝库内容注入 prompt，让入住者「记得」这些珍藏
- **双向珍藏**：我珍藏 ta 的话（我的宝库）+ ta 珍藏我的话（他的宝库），形成双向连接

## 二、我们的照片回忆

### 定位

保存聊天中发过的、用户想留下的图片——截图、表情包、手绘、生成图。

这不是图片管理工具，而是关系的视觉记录。每张图片背后都有一个时刻。

> 注意：当前版本不支持图片发送，照片回忆功能**设计先行，待图片功能实现后落地**。

---

### 第一版功能设计（待图片功能就绪后实现）

#### 保存触发方式

- 聊天页：收到/发出图片消息后，消息下方出现「📷 存入回忆」按钮
- 用户点击后，进入照片信息填写：
  - 标题（可选）
  - 备注（可选）
  - 时间（自动填入，可修改）
  - 归属入住者（默认当前入住者）

#### 数据结构草案

```js
// PhotoMemory
{
  id: string,
  charId: string,                // 关联的入住者
  imageData: string,             // base64 或 blob URL（受 localStorage 限制）
  title: string,
  note: string,
  occurredAt: string,            // YYYY-MM-DD
  createdAt: number,
  source: "chat" | "upload",
  sourceMessageId: string | null,
  pinned: boolean,
}
```

#### 存储限制说明

图片以 base64 形式存入 localStorage，**存储空间有限**，需要：
- 图片压缩到合理尺寸（建议宽度不超过 800px，压缩后不超过 200KB）
- 控制数量上限（建议每个入住者不超过 50 张）
- 超出上限时提示用户删除旧照片

---

### 展示入口

照片回忆的展示位置：

1. **「他的房间」中**：「我们的照片」区块，网格展示缩略图
2. **时间线中**：时间线事件可以关联一张照片
3. **便签墙中（未来）**：便签可以附图

---

### 后续扩展

- **图片标注**：在图片上标注重要的细节或情绪
- **照片和时间线联动**：创建时间线事件时可以附带一张照片
- **双方视角**：「我们的照片」展示时可以切换「我的视角」和「ta 的视角」
- **照片书**：将多张照片按时间线生成一本「回忆相册」导出
- **生成图片**：未来 AI 画图阶段，生成的入住者形象图片自动存入照片回忆

---

## 两个功能的关系

| | 我的宝库 | 他的宝库 | 我们的照片回忆 |
|--|---------|---------|--------------|
| 保存主体 | 用户 | 入住者（用户触发） | 用户 |
| 内容类型 | 入住者的文字 | 用户的文字 | 图片 |
| 归属维度 | 全局（不区分入住者） | 入住者维度 | 入住者维度 |
| 注入 prompt | 否 | 否（第一版） | 否 |
| 当前状态 | 已实现 | 未实现 | 未实现 |

三者共同构成**小家的珍藏系统**。
